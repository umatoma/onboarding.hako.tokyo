# 🚀 ITエンジニア オンボーディングポータル & ベストプラクティス研究

中途入社したITエンジニアが早期にチームに馴染み、実力を発揮できるようにするためのオンボーディングポータルと、その設計基盤となるベストプラクティスの調査・研究用リポジトリです。

> [!IMPORTANT]
> - **新メンバーの方へ**: `📅 オンボーディング・タイムライン` の5大フェーズに沿って進めながら、実績解除型の [🏆 オンボーディング・クエスト (docs/onboarding_quest.md)](docs/onboarding_quest.md) に挑戦してください。
> - **開発チーム・人事の方へ**: オンボーディングの設計思想や改善を行う際は、`🔍 オンボーディングのベストプラクティス調査` 内の資料を参照してください。

---

## 📅 オンボーディング・タイムライン & 進捗状況

中途入社メンバーのオンボーディングステップです。体系ガイドに基づき設計された5つのフェーズに従って、段階的に適応を進めていきます。各フェーズの具体的なタスクは [🏆 オンボーディング・クエスト](docs/onboarding_quest.md) にて管理・進行します。

### 🕒 1. 事前準備期 (〜Day 0)
*   **目標**: 入社初日からスムーズに開発に参加できる環境を整える
*   **タスク (受け入れチーム・人事による準備)**:
    - [ ] **機材準備**: 開発用PC・備品の手配および発送
    - [ ] **アカウント発行**: 主要ツール（GitHub, Slack, Jira, AWS/GCP等）の招待・ライセンス発行の申請
    - [ ] **受入調整**: 初日の歓迎ランチ、ウェルカムミーティング、メンター/バディの決定と日程調整
    - [ ] **初日タスク選定**: 初日の「Day 1 Commit」用タスク（Good First Issue等）の準備・選定
    - [ ] **ドキュメントの事前記入 (docs/配下)**:
        - [ ] [🛠️ setup.md](docs/setup.md) の `[要記入]` 箇所の設定
        - [ ] [🤝 working_agreement.md](docs/working_agreement.md) の `[要記入]` 箇所の設定
        - [ ] [🗺️ hard_skill_map.md](docs/hard_skill_map.md) の `[要記入]` 箇所の設定
        - [ ] [🏗️ architecture.md](docs/architecture.md) の `[要記入]` 箇所の設定
        - [ ] [🔄 development_process.md](docs/development_process.md) の `[要記入]` 箇所の設定
        - [ ] [👥 team_communication.md](docs/team_communication.md) の `[要記入]` 箇所の設定
        - [ ] [🚩 first_tasks.md](docs/first_tasks.md) の `[要記入]` 箇所の設定

### 🏁 2. 集中インプット期 (Day 1〜Week 1)
*   **目標**: 開発環境を構築し、チームに馴染み、最初のコミット（マージ）を行う
*   **タスク**:
    - [ ] [メンター・バディの紹介](docs/team_communication.md#メンター制度バディ制度)を受ける
    - [ ] [PCの初期設定・開発ツールのインストール](docs/setup.md#pcの初期設定)
    - [ ] [開発用リポジトリのクローンとローカルビルド・起動](docs/setup.md#開発用リポジトリのセットアップ)
    - [ ] **Quest 1**: 開発環境を起動せよ（`onboarding_quest.md` 内）
    - [ ] **Quest 2**: チームメンバーに自己紹介し、交流せよ
    - [ ] **Quest 3**: Day 1 Commit (初日のファーストマージ)の完了 (AI支援を活用)
    - [ ] **Quest 4**: 初めての「15分ルール」を適用せよ
    - [ ] **Quest 5**: 技術ギャップを自己評価し、学習計画を合意せよ

### 🛠️ 3. 業務適応期 (Week 2〜Month 1)
*   **目標**: チームのルールや他部署の連携方法を学び、技術ギャップを外部教材やAIで埋める
*   **タスク**:
    - [ ] [開発プロセスとチームルールの確認](docs/development_process.md)
    - [ ] [🤝 ワーキングアグリーメント](docs/working_agreement.md) の読み込み（15分ルールとAI/人間への質問の切り分け）
    - [ ] [🗺️ ハードスキルマップ & AI自学自習ガイド](docs/hard_skill_map.md) による技術ギャップの自己評価と学習計画の開始
    - [ ] **Quest 6**: ご近所さんとコーヒーチャットをせよ
    - [ ] **Quest 7**: AIバディと共にコードベースを探索せよ
    - [ ] **Quest 8**: 定例会議でアジェンダに沿って発言せよ
    - [ ] **Quest 9**: 初めての「コードレビュー」を実施せよ
    - [ ] **Quest 10**: チームの「バリュー」に沿った行動を1つSlackで共有せよ

### 🎯 4. 自走準備期 (Month 2〜Month 3)
*   **目標**: サポートを最小限に抑え、単独でタスクをリリースできる自走力を身につける
*   **タスク**:
    - [ ] **Quest 11**: 最初のメインタスクをリリースせよ
    - [ ] **Quest 12**: ペアプロの「ドライバー」を務めよ
    - [ ] **Quest 13**: ライブドキュメントに貢献せよ（古い情報の修正PR作成）
    - [ ] **Quest 14**: オンボーディングの「ふりかえり」を主催せよ
    - [ ] **Quest 15**: チーム内外へミニ勉強会（Tech Talk）を実施せよ

### 🌟 5. 定着フォロー期 (Month 3〜Month 6)
*   **目標**: 期待値のすり合わせを行い、長期的に活躍できる状態を定着させる
*   **タスク**:
    - [ ] 入社3ヶ月目の期待値すり合わせ面談（人事・マネージャー）
    - [ ] 入社半年後のフォローアップ面談およびアンケートへの回答

## 📂 開発用テンプレート (docs/)

新メンバーが業務に入る際、直接参照・更新する実務用ガイドラインとテンプレートです。

### 💻 環境構築 & 開発ルール
1.  **[🛠️ 開発環境セットアップガイド](docs/setup.md)**

2.  **[🔄 開発プロセス & チームルール](docs/development_process.md)**

### 👥 コミュニケーション & アグリーメント

5.  **[👥 チーム・コミュニケーションガイド](docs/team_communication.md)**

### 🏆 進捗管理 & 学習
6.  **[🏆 オンボーディング・クエスト (実績解除マップ)](docs/onboarding_quest.md) [NEW]**

7.  **[🗺️ ハードスキルマップ & AI自学自習ガイド](docs/hard_skill_map.md) [NEW]**
