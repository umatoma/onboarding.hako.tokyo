## 💬 2. コミュニケーションルール & チャンネル

### 基本姿勢
- **非同期コミュニケーション**: 議論や質問はできる限りパブリックなSlackチャンネルで行い、情報が属人化しないようにします。
- **テキストに残す**: ミーティングで決定した内容は、ConfluenceやGitHubのIssue/PRなどに必ず記録を残します。
- **気軽な相談 (雑談・質問)**: 悩んだら一人で抱え込まず、15分調べても解決しない場合はSlackでメンションを付けて気軽に質問してください。

### 主要な Slack チャンネル
| チャンネル名 | 用途 | 参加の目安 |
| :--- | :--- | :---: |
| `#team-[プロジェクト名]-all` | チーム全体の連絡・情報共有 | 必須 |
| `#team-[プロジェクト名]-dev` | 開発に関する技術的な相談・通知 | 必須 |
| `#dev-alerts` | エラー監視・デプロイ通知用チャンネル | 必須 |
| `#random` | 雑談・自己紹介・イベント等の案内 | 任意 |
| `#help-it` | 社内IT環境（PC・アカウント）に関するトラブル相談 | 必須 |

- **オンボーディングバディ**: **[要記入: バディ氏名]**
